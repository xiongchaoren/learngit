npm常用方法

1. 包使用者常用Command

1.1 包的初始化

  • npm init: 在当前目录下以用户引导的方式创建一个全新的package.json文件
  • -yes: 快速创建一个默认配置的package.json文件

1.2 包的下载安装

  • npm install: 安装当前目录下package.json文件里依赖项的包
  • packageName: 安装指定NPM包，不配置在package.json中
  • --save: 安装的包在生产环境(dependencies)
  • --save-dev: 安装的包在开发(DevDependencies)环境
  • -g: 安装在全局环境的包,可通过npm config ls查看安装位置，linux一般是在/usr/local/下
  • @version-number: 安装指定版本的包

1.3 卸载包

  • -g: 卸载全局包

1.4 更新包

  • -g: 更新全局包

1.5 查看NPM配置

  • npm config ls: 查看部分配置信息
  • -l: 查看全部配置信息

1.6 修改NPM配置的三种方法

  • 文件修改：修改.npmrc文件
  • 命令修改：$ npm config set <key> <value>
  • 追加命令修改：$ npm <command> <-config-key=value>

1.5 查看包的依赖关系

1.6 查看本地或全局依赖包和是否是最新版本

  • npm list: 查看本地包
  • -g: 查看全局包

2.1 四个步骤：

  1. 注册一个npm账号,也可以到npm官网注册

  2. 登陆npm账号: $ npm login

  3. 发布项目，在package.json目录下

  4. 退出登陆

3. package.json介绍

3.1 常见字段说明

  1. name：包名

  2. version：版本

  3. description：包用途介绍

  4. main：入口文件

  5. scripts：可执行命令行自定义脚本(或命令行)，npm run <action-key>执行

    scripts内置命令：start restart test stop

  6. repository：远程仓库，一般都是git

  7. keywords：包的关键词，方便搜索

  8. author：作者

  9. license：许可协议

  10. bugs：issues地址

  11. dependencies：生产环境依赖包

  12. devDependencies：开发环境依赖包
